check_as_mip_start(error_handler=None)[source]¶

Checks this solution is a valid mip start. To be valid, it must satisfy: (i) having at least one discrete variable (integer or binary) (ii) the value for integer/binary variables should be consistent with the type.

contains(dvar)[source]¶

Checks whether a decision variable is mentioned or not in the solution. Note that this predicate can also be used in the form “var in solution”, as the __contains_ method has been redefined for this purpose.

Parameters:dvar – The variable to check.
Returns:True if the variable is mentioned in the solution.
get_objective_value()[source]¶

Gets the objective value as defined in the solution. When the objective value has not been defined, a special value NO_SOLUTION is returned. You can check whether the objective has been set using has_objective().

Returns:The value of the objective as defined by the solution.
get_value(dvar)[source]¶

Gets the value of a solution variable in a solution. If the variable is not mentioned in the solution, returns 0 and does not raise an exception. Note that this method can also be used as solution[dvar] as the __getitem__ method has been overloaded.

Parameters:dvar – The decision variable for which we ask the solution value.
Returns:A floating-point number, the value of the variable in the solution.
