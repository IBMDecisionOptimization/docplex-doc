docplex.mp.context module¶

class docplex.mp.context.DOcloudContext(url=None, api_key=None)[source]¶

Bases: object

The context to solve models on DOcloud.

The context contains properties that alter the behavior of the DOcloud connector.

url¶

The DOcloud service URL.

api_key¶

The DOcloud service API key to use.

display_logs_after_run¶

If True, logs are downloaded and printed after the model is run.

run_deterministic¶

Specific engine parameters are uploaded to keep the run deterministic.

verbose¶

Makes the connector verbose.

timeout¶

The timeout for requests.

waittime¶

The wait time to wait for jobs to finish.

verify¶

If True, verifies SSL certificates.

log_requests¶

When True, the REST requests are logged.

exchange_format¶

The exchange format to use. When setting the format, you can use the following strings: “lp”. When getting the format, the property type is docplex.mp.format.ExchangeFormat.

has_credentials()[source]¶

Returns True if this context has valid credentials.

static make_context_from_python(pyconfig_module_path)[source]¶

Builds this context from a Python initialization file.

Recognized options have the same name as the attributes of DOcloudContext.

static make_from_config_parser(config, section_name=None)[source]¶

Reads this context from the given ConfigParser.

Config is read from section ‘DOcloudConnector’.

Recognized options have the same name as the attributes of DOcloudContext.

read_from_rcfile(filename)[source]¶

Reads this context from the given ressource file.

The specified ressource file contains name=value pairs. For example:

url = "https://docloud.service.com/job_manager/rest/v1"
api_key = "example api_key"
update_from_list(values)[source]¶

Updates this context from the list of properties specified as (name, value) tuples.
