check_as_mip_start(error_handler=None)[source]¶
Checks that this solution is a valid MIP start. To be valid, it must have:
  • at least one discrete variable (integer or binary), and
  • the value for integer/binary variables should be consistent with the type.
Parameters:error_handler – An instance of an error handler or None.
contains(dvar)[source]¶

Checks whether or not a decision variable is mentioned in the solution. Note that this predicate can also be used in the form var in solution, because the __contains_() method has been redefined for this purpose.

Parameters:dvar (docplex.mp.linear.Var) – The variable to check.
Returns:True if the variable is mentioned in the solution.
get_objective_value()[source]¶

Gets the objective value as defined in the solution. When the objective value has not been defined, a special value NO_SOLUTION is returned. You can check whether the objective has been set using has_objective().

Returns:The value of the objective as defined by the solution.
get_value(dvar_arg)[source]¶

Gets the value of a solution variable in a solution. If the variable is not mentioned in the solution, the method returns 0 and does not raise an exception. Note that this method can also be used as solution[dvar]() because the __getitem__() method has been overloaded.

Parameters:dvar-arg – A decision variable (docplex.mp.linear.Var) or a variable name (a string).
Returns:A floating-point number, the value of the variable in the solution.
has_objective()[source]¶
Returns:True if the solution defines an objective value.
is_empty()[source]¶

Checks whether the solution is empty.

Returns:True if the solution is empty; in other words, it has no defined objective and no variable value.
