docplex.mp.kpi module¶

class docplex.mp.kpi.KPI[source]¶

Bases: object

Abstract class for KPIs.

KPIs have a unique name.

KPIs are attached to a model instance and can compute a numerical value, using the compute() method.

Some KPIs require a valid solution of the model, while others do not. You can check whether a given KPI requires a solution by using requires_solution().

get_model()[source]¶

Returns the model instance on which the KPI is defined.

get_name()[source]¶

Returns the published name of the KPI.

The published name is a non-empty, unique string identifying the KPI.

requires_solution()[source]¶

Returns True if the KPI requires a valid solution.

KPIs based on decision expressions or variables require a successful solution to be computed. KPIs based on functions may not require a solution.
